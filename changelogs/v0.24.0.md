## 🔧 重构

### 消除 Items/Panel 渲染逻辑重复

**问题**：`renderPanel`（Items 模式）与 `CollapsePanel` 组件（Panel 模式）的渲染模板几乎完全相同——header 结构、icon 渲染、collapsible 逻辑、Transition 动画、forceRender CSS 隐藏——全部写了两遍（约 200 行重复代码）。

**修复**：

- 提取 `renderPanelNode()` 共享函数，Items 模式和 Panel 模式共用**唯一**的渲染模板
- 提取 `deriveCollapsibleState()` 纯函数，collapsible 派生状态的计算逻辑统一为一份
- 提取动画/样式常量（`CLOSED_HEIGHT`、`ICON_ROTATE_OPEN` 等），消除魔法字符串

```typescript
// 共享面板渲染 — 两模式共用
function renderPanelNode(options: PanelRenderOptions): VNode {
  /* ... */
}

// 共享状态计算
function deriveCollapsibleState(disabled: boolean, collapsible?: CollapsibleType): CollapsibleState {
  /* ... */
}
```

### CollapsePanel 独立文件

将 `CollapsePanel` 组件从 `Collapse.tsx` 分离到 `CollapsePanel.tsx`，共享内部 API 通过 `Collapse.tsx` 的 export 暴露：

```
components/collapse/
├── Collapse.tsx          # 组件 + 共享内部 API
├── CollapsePanel.tsx     # Panel 子组件（新建）
├── index.ts
├── types.ts
└── ...
```

### renderItemPanel 提升到 setup 级别

`renderItemPanel` 是纯函数，原本定义在 render 函数内部导致每次渲染重新创建。已提升到 `setup()` 级别。

## ✨ 新特性

### 键盘无障碍

Header 元素新增键盘事件处理，符合 WAI-ARIA 规范：

- `onKeydown` 处理 `Enter` 和 `Space` 键展开/收起面板
- `tabindex` 管理：禁用面板 `-1`，正常面板 `0`

```tsx
<div
  onKeydown={(e) => { if (e.key === 'Enter' || e.key === ' ') { e.preventDefault(); onToggle() } }}
  tabindex={isDisabled ? -1 : 0}
  role="button"
  aria-expanded={isOpen}
  aria-disabled={isDisabled}
>
```

### Item 级别 classNames/styles

`CollapseItem` 类型新增 `classNames` 和 `styles` 字段，支持面板级别的语义化样式定制，优先级高于 Collapse 级别：

```typescript
const items: CollapseItem[] = [
  {
    key: '1',
    label: '定制面板',
    children: '...',
    classNames: { header: 'my-custom-header' },
    styles: { header: { fontWeight: 600 } },
  },
]
```

## 🐛 修复

### 类型安全

| 修复项                 | 修复前                                | 修复后                                                  |
| ---------------------- | ------------------------------------- | ------------------------------------------------------- |
| Context 类型           | `inject<any>(COLLAPSE_CONTEXT_KEY)`   | `inject<CollapseContext \| null>(COLLAPSE_CONTEXT_KEY)` |
| `CollapseItem.style`   | `Record<string, any>`                 | `CSSProperties`                                         |
| `ExpandIconProps` 字段 | 全部 `?` 可选                         | `isActive`、`panelKey` 改为必填                         |
| `canClickIcon` 推导    | 冗余 `!== 'disabled'` 比较触发 TS2367 | 简化 `!isDisabled`（已蕴含排除逻辑）                    |

### CSS 修复

- **`:not()` 嵌套选择器兼容性**：`.hmfw-collapse-header:hover:not(.hmfw-collapse-item-disabled .hmfw-collapse-header)` → `.hmfw-collapse-item:not(.hmfw-collapse-item-disabled) .hmfw-collapse-header:hover`
- **删除冗余样式**：`.hmfw-collapse-icon-position-end .hmfw-collapse-header { flex-direction: row }`（`row` 是 flex 默认值）
