## [0.23.0] - 2026-07-14

Empty 组件从 API 设计、健壮性与边界条件、设计模式与架构、可读性与可维护性四个维度全面代码审查与增强。

### 🔧 重构

- **Empty props 类型规范化**：内联 props 提取为 `emptyProps` 并使用 `satisfies Record<keyof EmptyProps, any>` 模式，强制运行时 props 与 `EmptyProps` 接口单一真值源同步，杜绝双源头漂移
- **补齐可选属性默认值**：`imageStyle`、`classNames`、`styles` 补 `default: undefined`，满足 satisfies 对可选字段的匹配要求
- **图片渲染逻辑去重**：自定义插槽 / URL 图片 / 预设插画三个分支原本各自重复 `hmfw-empty-image` 容器，现统一先计算 `imageContent` 再由单一 wrapper 包裹
- **消除 `as any`**：改用 `Object.assign` + 复合类型 `EmptyType` 挂载 `PRESENTED_IMAGE_DEFAULT` / `PRESENTED_IMAGE_SIMPLE` 预设常量，静态属性带完整类型
- **收紧内部类型**：`imageNode` 从 `unknown` 收紧为 `VNode | null`，`sizeStyle` / `mergedImageStyle` 使用 `CSSProperties`

### ✨ 新特性

- **`img` 替代文本无障碍增强**：图片 URL 模式下 `alt` 从写死的 `"empty"` 改为优先取字符串 `description`（无字符串描述时回退 `"empty"`），与 AntD v6 对齐，利于屏幕阅读器
- **`#description` 插槽优先级高于 `description={false}`**：`showDescription` 逻辑调整为 `!!slots.description || props.description !== false`，提供了描述插槽时始终展示，即使 `description` 设为 `false`

### 💥 Breaking Changes

- **`imageStyle` 类型变更**：从 `Record<string, string>` 改为 `CSSProperties`，与 `styles.image`、文档及 AntD 对齐，允许数值型样式属性（对绝大多数用法向后兼容）

### 📝 文档

- **新增 2 个 Demo**：
  - `EmptyCustomImage`：自定义图片（图片 URL、`#image` 插槽、`image={false}` 隐藏图片）
  - `EmptyDescription`：描述控制（`description={false}` 隐藏、`#description` 富文本插槽、插槽覆盖 `description={false}`）
- **API 文档修正**：`imageStyle` 类型标注统一为 `CSSProperties`；补充 `alt` 取值规则与 `description={false}` 和插槽的优先级说明
- **Demo 覆盖率提升**：现覆盖 Empty 全部对外能力（默认/简约插画、图片 URL、自定义图片插槽、隐藏图片、自定义/富文本描述、描述隐藏、底部操作、尺寸控制、暗黑模式、语义化 classNames/styles）

### 🧪 测试

- **测试从 18 → 21**：新增 3 个用例
  - 字符串 `description` 作为 `img` 的 `alt`
  - 非字符串 `description` 时 `alt` 回退 `"empty"`
  - `#description` 插槽优先级高于 `description={false}`

### 📊 质量提升

- **规范符合**：违反 satisfies props 规范 → 完全符合
- **类型安全**：消除 1 处 `as any`、1 处 `unknown`
- **无障碍**：修复 `img alt` 写死回归
- **测试覆盖**：18 → 21 (+17%)
- **文档完整**：7 → 9 Demo (+29%)，覆盖全部对外能力
